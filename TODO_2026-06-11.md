## 2026-06-11 — Writing Editor, Book Cards & Pricing Model

### Fixed copy/paste in the writing editor

- Root cause: circular update loop in QuillEditor — `onUpdate` emitted both `textChange` and `update:content`, which caused `WritingPage` to set `editorContent` twice, triggering the content watcher to call `setContent` and reset the editor state on every keystroke.
- Fix: Added a `suppressContentWatch` ref flag in QuillEditor.vue that skips the content watcher for one tick after an internal edit. Simplified WritingPage to bind only `@update:content` instead of dual events.

### Auto-publish first chapter when a book is published

- **Frontend** (`usePublishFlow.ts`, `WritingPage.vue`): `confirmPublish` now accepts an optional `chapters` array. After publishing the book, it finds the first chapter (lowest order) and calls `updateChapter` to set its status to PUBLISHED.
- **Backend** (`books.service.ts`, `books.module.ts`): Added `publishFirstChapter()` private method called when book status transitions to PUBLISHED. Injected `Chapter` repository into the books module.

### Redesigned book cards (Explore + Home)

- **ExploreBookCard.vue** — Full rewrite:
  - Status tags: *New* (sky blue, ≤14 days), *Popular* (amber, ≥100 reads / ≥20 likes / ≥4.0 rating), *Exclusive* (violet + Crown icon, isPremium), price badge (rose outline, isPurchasable)
  - 5-star rating display with `Math.round(rating)` and numeric score
  - Deterministic gradient fallback covers (8 schemes hashed by category + id)
  - Glass-morphism badges with backdrop-blur, hover overlay with "View Details" CTA
  - Navigates to `/book-detail/:id`

- **Book_collection.vue** (Home carousel) — Polished:
  - `aspect-[3/4]` covers, 260px card width, dot pill pagination
  - Rating stars, genre badges, hover lift + shadow
  - Skeleton loading states, error/retry state
  - Reveal-on-scroll animation with IntersectionObserver

### Extended ExploreBook type with pricing model fields

- `src/types/exploreBook.ts` — Added required fields: `isFree`, `isPurchasable`, `isPremium`, `price`, `readCount`, `likeCount`
- `src/services/exploreApi.ts` — `mapToExploreBook` now maps all pricing fields with `Number()` / `Boolean()` coercion
- `src/data/mockExploreBooks.ts` — 12 mock entries with varied pricing models (free, purchasable, premium) for testing

### Fixed 400 Bad Request on pricing save (SettingsDrawer)

- **Backend** (`update-book.dto.ts`): Added `@Type(() => Number)` before `@IsNumber()` on `price`, and `@Type(() => Boolean)` before `@IsBoolean()` on `isFree`, `isPurchasable`, `isPremium`. Without these, `class-transformer` never coerces the raw JSON body values, so `class-validator` sees strings and rejects with 400.
- **Frontend** (`SettingsDrawer.vue`): Added `Number(localPrice) || 0` in the `savePricing` emit as a safety net.

### New books now default to free

- **Backend entity** (`book.entity.ts`): Changed `isFree` column default from `false` to `true`.
- **Backend service** (`books.service.ts`): `create()` now explicitly sets `isFree: true`, `price: 0`, `isPurchasable: false`, `isPremium: false`.
- Users can change to purchasable/premium later via the Settings drawer on the Writing page.
